1. 基本概念: HTTP DNS TCP UDP 网卡驱动程序 物理硬件网卡

2. HTTP 是基于 TCP 协议传输的

3. TCP报文有 端口号（发给哪个应用比如qq还是微信）、序号（数据包要正确顺序组装）、窗口大小（TCP做流量控制的，还有拥塞机制不理解）

4. TCP 三次握手  -  主要确认双方都有发送和接收的能力

5. 网络包报文（网路链路层）： MAC头部 \ IP头部 \ TCP头部 \ HTTP报文 , 其中在TCP头部之中就有很多数据用于握手、拥塞、序号的东西

6. 应用层 - HTTP数据

传输层 - TCP头部

网络层 - IP头部

链路层 - 以太网头部
